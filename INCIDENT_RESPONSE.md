# Incident Response

Critical incidents include active data exposure, compromised credentials, payment/identity fraud, or immediate safety impact. High incidents include exploitable authorization failures, sensitive-data leakage, or major integrity failures.

Contain → preserve evidence → assess data/users/providers → remove access or keys → notify owner → remediate → verify → document → review. Never place secrets in tickets or frontend code.

Production launch requires named owners, secure contact paths, processor notification procedures, Korean breach-notification legal review, backup restoration tests, and an incident exercise.
