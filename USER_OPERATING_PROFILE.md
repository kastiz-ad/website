# Kastiz ONE User Operating Profile

The launch profile is progressive, optional and human-readable. ONE asks for mission information first and offers to remember selected non-sensitive preferences only after an unchecked user choice.

Implemented categories: General, Travel, Food, Shopping, Housing, Education, Business and Accessibility. Stored values are validated against a category-specific allowlist and labeled `device-local`.

Current principle: **Ask once. Remember with permission. Prepare automatically. Execute only with approval.**

Users manage saved values at `profile.html`: pause memory, review, edit, delete a field, clear a category, clear the full profile or export a text summary.
