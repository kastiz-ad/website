# Analytics Privacy

Optional analytics defaults off. Reject and Accept are equally accessible; users can reopen choices from Settings. Consent category, version and timestamp are stored. Withdrawal blocks future optional events and updates GA consent to denied.

Analytics excludes raw mission text, names, emails, phone numbers, precise location, documents, passport/visa/health/payment/child/emergency data, uploaded contents and provider-sensitive data. No fingerprinting is used. Anonymous IDs are random and created only after analytics consent.

Analytics events must never be used as an execution approval record.
