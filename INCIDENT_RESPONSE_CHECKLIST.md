# Incident Response Checklist — Draft

1. Preserve evidence and record the incident time without copying sensitive payloads into chat or tickets.
2. Contain access: revoke affected sessions, provider tokens, and server secrets; disable affected feature flags.
3. Assess users, records, providers, environments, and approval/execution attempts affected.
4. Rotate compromised credentials and verify RLS/policies before restoration.
5. Restore from a verified backup when integrity is uncertain.
6. Obtain qualified Korean legal/privacy advice for notification duties and timing.
7. Document root cause, corrective actions, owners, and a follow-up test.
