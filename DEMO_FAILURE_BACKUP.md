# Demo failure backup

- Wi-Fi fails: open `offline-demo/index.html` directly in Chrome/Edge. State clearly that it is an offline demonstration with no live-data claim.
- Public API fails: continue; the UI should show `fallback_demo` / demonstration recommendation. Do not call it live.
- Login unavailable: use Login to explain gradual access, then open Early Access. No account is expected in this build.
- Mobile display fails: rotate once, reload, then switch to the prepared 1440×900 desktop route. Do not zoom during the presentation.
- Results do not load: return home and retry `?demo=1`; if still blocked, use the offline demo and narrate the approval checkpoint.
- Never improvise a booking, payment, customer, partner or live availability claim.
