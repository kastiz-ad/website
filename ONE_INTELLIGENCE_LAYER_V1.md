# ONE Intelligence Layer V1

Status: **prototype, not live**. No calendar, email, traffic, location, booking, payment or provider integration is connected.

## Philosophy

ONE uses only approved, permitted signals to reduce mental load like a thoughtful secretary. It does not surveil, advertise, repeatedly interrupt or act automatically. Repetition is evidence, not certainty; one event remains low confidence. Suggestions require medium or high confidence, context relevance, privacy approval, fatigue clearance and diversity review.

## Pipeline

Approved signal → Behavior Pattern Engine → Context Awareness → Need Prediction → Diversity and Fatigue Control → Confidence → Privacy Guard → one suggestion → accept/dismiss/ignore → feedback learning.

The homepage renders the engine result. Intelligence logic does not live in homepage components. The main search remains dominant and completely usable when suggestions load, disappear or fail.

## Future adapters

Calendar, email state, mission history, Live Intelligence, location and workspace availability have interface boundaries only. Context values must be labeled `LIVE`, `ESTIMATED`, `DEMO` or `UNAVAILABLE`; unavailable live context is never invented.

## Approval

Accept means “Yes, help me with this.” It enters the existing preparation and approval path. It never means send, purchase, reserve, book, pay, contact or submit.
